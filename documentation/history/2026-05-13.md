# 2026-05-13 — Obsidian community review fixes

Addressed the warnings listed on https://community.obsidian.md/account/plugins/dataview-serializer for review of commit 070bd18 (v2.6.1).

## Source code

- `manifest.json`: bumped `minAppVersion` from `0.15.0` to `1.4.10` to cover the `ButtonComponent.setIcon`, `ButtonComponent.setTooltip`, and `AbstractInputSuggest` APIs already used in the codebase.
- `package.json`: added `@codemirror/view` and `@codemirror/state` as direct dependencies (they are imported by `refresh-button-extension.ts`). Bumped `packageManager` to `bun@1.3.14`.
- `src/app/plugin.ts`: added descriptions to the `eslint-disable` directive for the untyped `app.plugins` access; renamed commands to drop the plugin-name prefix (`Insert Dataview serializer block` → `Insert query block`; `Remove all Dataview serializer queries from current file` → `Remove all queries from current file`); awaited the `saveSettings()` call when loading settings; removed the redundant `_file as TFile` cast.
- `src/app/utils/serialize-query.fn.ts`: added a description to the `no-explicit-any` directive over the `vault.config` access.
- `src/app/utils/serialize-dataviewjs-query.fn.ts`: replaced `setTimeout` / `clearTimeout` with `activeWindow.setTimeout` / `activeWindow.clearTimeout` for popout-window compatibility, and ensured the `.catch` handler always rejects with an `Error` instance.
- `src/app/utils/dataviewjs-proxy.ts`, `src/app/utils/literal-to-string.fn.ts`: removed unnecessary `as Literal` / `as Literal[][]` assertions.
- `src/app/refresh-button-extension.ts`: replaced `document.createElement('span'|'button')` with `createSpan()` / `createEl('button', ...)` for popout-window compatibility; widened `createRefreshButton`'s `onClick` type to accept `void | Promise<void>`; wrapped the inline-query click handler in a `void (async () => {...})()` IIFE so the `addEventListener` callback returns `void`; replaced the two `console.error(...)` calls with the project `log()` helper.
- `src/app/settings/settings-tab.ts`: converted the async `onClick` callbacks in `doSearchAndRemoveList` to non-async handlers that fire-and-forget the inner async work via `void (async () => {...})()`, and switched the `forEach` to a block body so its return value is explicitly `void`.
- `src/utils/log.ts`: suppressed all `console.*` calls in the shipped bundle (kept the function and switch shape intact behind comments for easy re-enabling).
- `src/utils/log.spec.ts`: rewrote tests to assert that no console output is produced and that the function does not throw.
- `src/test-setup.ts`: prefixed the `mock.module('obsidian', ...)` call with `void` to satisfy the no-floating-promises rule.

## Styles

- `src/styles.src.css`: removed the `color: ... !important;` declaration on `.dvs-results-start-line, .dvs-results-end-line` and bumped specificity with a doubled-class selector (`.dvs-results-start-line.dvs-results-start-line, ...`) to override CodeMirror's line color without using `!important`.

## CI / release

- `.github/workflows/ci.yml`, `.github/workflows/release.yml`: replaced `bun-version: latest` with `bun-version-file: package.json` so CI uses the pinned `packageManager` version and release builds become reproducible.
- The release workflow already attaches `main.js`, `manifest.json`, `styles.css` only and generates GitHub artifact attestations; the next release should clear the "additional files" and "missing attestation" warnings.

## Notes for manual verification

The plugin runs in Obsidian's GUI and cannot be self-verified by an agent. When testing manually, confirm:

- The two renamed commands (`Insert query block`, `Remove all queries from current file`) appear under the **Dataview Serializer** group in the command palette with the new names.
- Refresh buttons on block and inline queries still render and trigger a refresh.
- The "Add folder" / "Remove" buttons in **Settings → Dataview Serializer** (ignore-list section) still save changes correctly.
- DataviewJS queries with timeouts still abort cleanly.
